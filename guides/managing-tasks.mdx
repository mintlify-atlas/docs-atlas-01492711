Tasks are the core units of work in your task management app. Learn how to effectively create, update, and manage tasks to keep your team on track.

## Creating a New Task

<Steps>
  <Step title="Navigate to Your Team">
    From the main dashboard, tap **View** on the team card where you want to add a task. This opens the team's task screen.
  </Step>

  <Step title="Open the Task Creation Dialog">
    Tap the black floating **Add** button (with a plus icon) at the bottom right of the screen. A task creation dialog will appear.
  </Step>

  <Step title="Enter Task Title">
    In the **Task Title** field, enter a clear, concise name for your task. This should describe what needs to be done.
    
    <Tip>
      Use action verbs to make tasks clear: "Design homepage", "Review budget", "Update documentation"
    </Tip>
  </Step>

  <Step title="Add Task Description">
    Use the **Description** field to provide detailed information about the task:
    - What needs to be accomplished
    - Any specific requirements or constraints
    - Links to relevant resources
    - Expected deliverables
    
    The description field supports multiple lines for detailed explanations.
  </Step>

  <Step title="Assign a Responsible Person">
    Tap the **Responsible** dropdown to select a team member who will be accountable for this task.
    
    The dropdown shows all current team members. Only team members can be assigned to tasks.
    
    <Note>
      Assigning a responsible person helps ensure accountability and clarity about task ownership.
    </Note>
  </Step>

  <Step title="Set Task Status">
    Choose the current status of the task by tapping one of three status indicators:
    
    - **No Iniciado** (Not Started) - Red indicator: Task hasn't begun
    - **En desarrollo** (In Progress) - Orange indicator: Task is actively being worked on
    - **Finalizado** (Completed) - Green indicator: Task is complete
    
    The selected status will be highlighted with a colored background.
  </Step>

  <Step title="Set Start and End Dates">
    Define the task timeline:
    
    **Start Date**: Tap the start date field and select when work should begin
    **End Date**: Tap the end date field and select the task deadline
    
    Both dates open a calendar picker for easy selection. Dates help team members prioritize work and track progress.
  </Step>

  <Step title="Save the Task">
    Review all task details and tap the green **Save** button. The task will be added to the team's task list and the assigned team member will be able to see it.
  </Step>
</Steps>

## Viewing Task Details

<Accordion title="How to view assigned member information">
  To see who is responsible for a task:
  
  1. Navigate to the team's task screen
  2. Tap on any task in the list
  3. A dialog will display the name of the assigned team member
  
  If no one is assigned, you'll see a message indicating no member is assigned to the task.
</Accordion>

## Editing Existing Tasks

<Steps>
  <Step title="Locate the Task">
    From the team screen, find the task you want to edit in the task list under the "Tasks" section.
  </Step>

  <Step title="Open the Edit Dialog">
    Tap the **Edit** icon (pencil icon) on the right side of the task. This opens the task update dialog with all current task information pre-filled.
  </Step>

  <Step title="Modify Task Details">
    Update any of the following fields:
    
    - **Title**: Change the task name
    - **Description**: Update or add more details
    - **Start Date**: Adjust the start date using the calendar picker
    - **End Date**: Change the deadline
    - **Status**: Update to reflect current progress
      - No Iniciado (Not Started)
      - En desarrollo (In Progress)
      - Finalizado (Completed)
  </Step>

  <Step title="Save Changes">
    After making your edits, tap the green **Update** button to save changes, or tap **Cancel** to discard modifications.
    
    The task list will automatically refresh to show your updates.
  </Step>
</Steps>

<Note>
  Only tasks assigned to you are shown by default when viewing "My Tasks". To see all team tasks, view them from the team screen.
</Note>

## Deleting Tasks

<Steps>
  <Step title="Find the Task to Delete">
    Navigate to the team screen and locate the task in the task list.
  </Step>

  <Step title="Tap the Delete Icon">
    Tap the circular pink **Delete** button (trash icon) on the right side of the task.
  </Step>

  <Step title="Confirm Deletion">
    A confirmation dialog will appear with a warning:
    
    *"Do you want to delete this task? Remember that the deleted task cannot be recovered."*
    
    - Tap **Accept** to permanently delete the task
    - Tap **Cancel** to keep the task
  </Step>
</Steps>

<Warning>
  Task deletion is permanent and cannot be undone. Ensure you really want to delete a task before confirming.
</Warning>

## Task Organization Strategies

<AccordionGroup>
  <Accordion title="Using Status Effectively">
    Keep your task statuses up-to-date for better team visibility:
    
    **Not Started (Red)**
    - Tasks that are planned but haven't begun
    - Use this for backlog items
    - Helps identify available work
    
    **In Progress (Orange)**
    - Tasks currently being worked on
    - Update to this status when you begin work
    - Prevents duplicate effort
    
    **Completed (Green)**
    - Finished tasks that meet acceptance criteria
    - Update immediately when done
    - Provides visibility into team accomplishments
  </Accordion>

  <Accordion title="Date Management Best Practices">
    Set realistic dates to keep your team on track:
    
    - **Buffer time**: Add extra time for unexpected delays
    - **Dependencies**: Consider task dependencies when setting dates
    - **Review regularly**: Update dates if priorities or requirements change
    - **Clear deadlines**: Set specific end dates, not just "sometime next week"
  </Accordion>

  <Accordion title="Writing Effective Task Titles">
    Good task titles are:
    
    **Specific**: "Design login page mockup" vs "Design stuff"
    **Actionable**: Start with a verb - "Update", "Create", "Review"
    **Concise**: Keep under 50 characters
    **Clear**: Anyone should understand what needs to be done
    
    **Examples of good titles:**
    - "Review Q2 marketing budget"
    - "Fix login authentication bug"
    - "Prepare client presentation slides"
  </Accordion>

  <Accordion title="Using Descriptions Effectively">
    Leverage the description field to provide context:
    
    - **Acceptance criteria**: What defines "done"?
    - **Resources**: Links to documents, designs, or references
    - **Constraints**: Time limits, budget restrictions, technical requirements
    - **Notes**: Important context or decisions made
    
    A well-written description reduces back-and-forth questions and helps the assignee complete the task efficiently.
  </Accordion>
</AccordionGroup>

## Viewing Your Personal Tasks

To see all tasks assigned to you across all teams:

1. From the main dashboard, tap the **Assignment** icon (clipboard icon) in the top app bar
2. This opens your personal task view showing only tasks where you are the responsible person
3. Tasks are organized by team

<Tip>
  Use the personal task view to focus on your individual workload without the distraction of unassigned team tasks.
</Tip>

## Task Filters and Views

**Team View**: Shows all tasks for a specific team, regardless of assignment
- Access from the team screen
- Useful for team leaders and project managers
- Provides full team visibility

**Personal View**: Shows only your assigned tasks across all teams
- Access from the assignment icon on the dashboard
- Helps you focus on your responsibilities
- Organized by team for context

## Common Task Management Workflows

<Steps>
  <Step title="Daily Task Review">
    Start each day by:
    1. Opening your personal task view
    2. Reviewing tasks due today or overdue
    3. Updating status of tasks you worked on
    4. Prioritizing your day's work
  </Step>

  <Step title="Weekly Team Sync">
    During team meetings:
    1. Open the team task view
    2. Review all in-progress tasks
    3. Update statuses as a team
    4. Reassign tasks if needed
    5. Create new tasks for upcoming work
  </Step>

  <Step title="Project Completion">
    When wrapping up a project:
    1. Mark all completed tasks as "Finalizado"
    2. Review any remaining tasks
    3. Update or delete outdated tasks
    4. Document lessons learned in task descriptions
  </Step>
</Steps>

## Troubleshooting

**Can't save a task?**
- Ensure the task title is not empty
- Check that you've selected a responsible person
- Verify dates are valid (end date after start date)

**Task not appearing in list?**
- Check you're viewing the correct team
- Pull down to refresh the task list
- Verify the task was saved successfully

**Can't edit a task?**
- Ensure you're a member of the team
- Check your internet connection
- Try closing and reopening the task screen
