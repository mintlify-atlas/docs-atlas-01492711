Notifications help you stay informed about task deadlines, updates, and team activities. Learn how to set up and manage notifications effectively.

## Notification System Overview

The task management app includes a notification system that helps you:

- Track upcoming task deadlines
- Monitor task assignments
- Stay updated on task status changes
- Never miss important project milestones

<Note>
  The app uses **flutter_local_notifications** to deliver timely alerts directly to your device, even when the app is not actively open.
</Note>

## Accessing Notifications

### Notification Screen

To view your notification history:

1. From the main dashboard, navigate to the notifications section
2. You'll see a list of all tasks with their current status
3. Each notification displays:
   - Task name
   - Task description
   - Completion status (check icon for completed, pending icon for incomplete)

**Visual Indicators:**
- **Green check icon**: Task is marked as completed
- **Red pending icon**: Task is incomplete or in progress

## Setting Up Notifications

<Steps>
  <Step title="Enable Notification Permissions">
    When you first use the app, you'll be prompted to allow notifications. Tap **Allow** to enable the notification system.
    
    **Why this matters:**
    - Receive timely alerts about tasks
    - Get reminders even when the app is closed
    - Stay synchronized with team updates
  </Step>

  <Step title="Configure Device Settings">
    Ensure your device allows notifications from the task management app:
    
    **For Android:**
    1. Open device Settings
    2. Navigate to Apps & Notifications
    3. Find your task management app
    4. Enable "Show notifications"
    5. Set notification importance to "High" or "Urgent"
    
    **For iOS:**
    1. Open device Settings
    2. Scroll to the task management app
    3. Tap Notifications
    4. Enable "Allow Notifications"
    5. Choose alert style (Banner, Alert, etc.)
  </Step>

  <Step title="Set Task End Dates">
    For tasks where you want reminders:
    
    1. Create or edit the task
    2. Set a specific end date using the date picker
    3. Save the task
    
    The notification system uses end dates to determine when to send deadline reminders.
  </Step>

  <Step title="Monitor Notification Center">
    Check the in-app notifications screen regularly to:
    - Review all your tasks
    - See completion status at a glance
    - Track overall progress
  </Step>
</Steps>

**Limited or No Permissions:**

The app uses **flutter_local_notifications** plugin to provide:

- Notifications are generated on your device
- No internet required for delivery
- Works even when the app is closed

**Scheduled Notifications**
- Reminders triggered at specific times
- Based on task end dates

- Task titles and descriptions
- Status information
- Actionable from notification shade

  flutter_local_notifications provides cross-platform notification support, ensuring consistent behavior on both Android and iOS devices.

## Managing Notification Preferences

### Notification Timing

To control when you receive notifications:

**Best Practices:**
- Set task end dates for automatic deadline reminders
- Use realistic deadlines to avoid notification fatigue
- Review and update task dates as priorities change
- Mark completed tasks as "Finalizado" to stop reminders

### Reducing Notification Overload

<Tip>
  If you're receiving too many notifications:
  
  1. Update completed tasks to "Finalizado" status
  2. Remove or adjust unrealistic end dates
  3. Focus on fewer, higher-priority tasks
  4. Consider device-level Do Not Disturb settings for specific times
</Tip>

## Troubleshooting Notifications

**Not receiving notifications?**
- Check device notification permissions for the app
- Ensure notifications aren't blocked in device settings
- Verify the app has battery optimization exceptions
- Restart the app to reinitialize notification service

**Too many notifications?**
- Update completed tasks to "Finalizado" status
- Review and adjust task end dates
- Enable Do Not Disturb during focus time
- Consider device-level notification batching

**Notifications not showing task details?**
- Ensure the app is updated to the latest version
- Check that task descriptions are saved properly
- Verify notification permissions allow full detail display

**Delayed notifications?**
- Check device battery optimization settings
- Ensure the app isn't being aggressively closed
- Verify your device has adequate resources
- Consider allowing the app to run in background

<Warning>
  Some Android devices aggressively manage battery by limiting background processes. You may need to whitelist the app in your device's battery settings for reliable notifications.
</Warning>

## Advanced Notification Settings

### Device-Specific Optimization

**Android Manufacturers:**

Some Android devices (Samsung, Xiaomi, Huawei) require additional steps:

1. **Battery Optimization:** Exclude the app from battery optimization
2. **Autostart:** Enable autostart for the app
3. **Background Activity:** Allow background data usage
4. **Notification Channels:** Configure app notification channels

**iOS Devices:**

1. **Screen Time:** Ensure app isn't restricted
2. **Focus Modes:** Configure to allow task notifications
3. **Notification Summary:** Decide if time-sensitive

<Tip>
  For the most reliable notifications, set the task management app as a "priority" or "essential" app in your device settings.
</Tip>
